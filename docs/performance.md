## Производительность (заметки)

Этот файл — справочные наблюдения по потенциальным узким местам и безопасным оптимизациям.

### Импорт (выгрузка 1С → SQLite)

- Узкое место обычно в количестве `ET.parse` и чтении многих файлов с диска (`shared/xml_parser.py` + `admin_tool/db_manager.py`).
- В `db_manager.py` уже применены ускоряющие PRAGMA на время загрузки (WAL, synchronous=OFF, cache_size, temp_store).
- Оптимизации делаются только после измерений на реальной большой выгрузке; не усложняем код «на всякий случай».

### MCP runtime (запросы к SQLite)

- В `server/tools.py` есть кэш соединений SQLite и инвалидция по `mtime` файла БД — это снижает накладные расходы на повторные запросы.
- `search_code` при фильтрах/спецсимволах переходит на `LIKE` по полному тексту кода; для больших БД это может быть тяжелее FTS. Дальнейшие оптимизации — только по реальным кейсам и метрикам.

### Про токены / размер ответов

- Ответы с подробной структурой форм могут быть «тяжёлыми» для ИИ по токенам. Если понадобится, можно добавить режимы «кратко/подробно» в инструменты (отдельная задача, без влияния на схему БД).
