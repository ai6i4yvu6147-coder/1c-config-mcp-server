=============================================================================
1C-Config MCP Server
=============================================================================

  Для ИИ и разработчиков: см. README_AI.md (базы создаются заново, миграций нет).

ОПИСАНИЕ:
  Инструмент для работы с конфигурациями 1С через любой ИИ-ассистент с поддержкой MCP.
  Позволяет анализировать код, искать объекты, изучать структуру.

СОСТАВ ПРОЕКТА:
  - admin_tool/     - Приложение управления базами и проектами (GUI)
  - server/         - MCP-сервер для подключения к ИИ-клиенту
  - shared/         - Общий код (парсер XML, ProjectManager)
  - databases/      - Папка с базами данных конфигураций (создаётся при первом запуске)
  - projects.json   - Список проектов и баз; активный проект задаёт, какие БД доступны MCP

=============================================================================
БЫСТРЫЙ СТАРТ:
=============================================================================

1. СОЗДАНИЕ БАЗЫ ДАННЫХ

   1.1. Запустите приложение управления (admin_tool).
   1.2. Создайте проект (если ещё нет) и сделайте его активным.
   1.3. Нажмите «Создать новую БД» (или добавьте базу в проект).
   1.4. Укажите путь к Configuration.xml из выгрузки конфигурации 1С.
   1.5. Дождитесь завершения сборки. База появится в списке баз проекта.

2. ПОДКЛЮЧЕНИЕ К ИИ-КЛИЕНТУ (MCP)

   2.1. Закройте приложение ИИ с поддержкой MCP (если открыто).

   2.2. Откройте файл конфигурации MCP вашего приложения.
        Например, для Claude Desktop: Win+R → %APPDATA%\Claude → claude_desktop_config.json
        Для другого клиента — см. его документацию.

   2.3. Добавьте MCP-сервер (замените ПУТЬ на реальный путь к папке проекта):

   {
     "mcpServers": {
       "1c-config": {
         "command": "ПУТЬ\\Server\\1c-config-server.exe"
       }
     }
   }

   Пример:
   "command": "C:\\projects\\1c-config-mcp\\Server\\1c-config-server.exe"

   ВАЖНО: Используйте двойные слэши \\ в пути.

   2.4. Сохраните конфиг и запустите ИИ-приложение.

3. ИСПОЛЬЗОВАНИЕ

   Список доступных проектов и баз возвращает инструмент active_databases.
   Указывайте project_filter по имени проекта при вызове search_code, find_object и др.

   Примеры запросов в ИИ-клиенте:
   - «Покажи список общих модулей»
   - «Найди где используется РегистрыСведений.ФТ_БизнесПроцессы»
   - «Покажи процедуры модуля ФТ_Бюджетирование»
   - «Найди код функции ПараметрыДанныхБюджета»

ПЕРЕКЛЮЧЕНИЕ МЕЖДУ БАЗАМИ:
  Активные базы задаются в projects.json: отметьте нужный проект как активный
  в приложении admin_tool. MCP-сервер при следующем запросе подхватит изменения
  (перечитывает projects.json при изменении файла).

=============================================================================
ПОРТАТИВНОСТЬ:
=============================================================================

  Папку можно копировать на другой компьютер или перемещать.
  После перемещения обновите путь к 1c-config-server.exe в конфиге MCP-клиента
  и перезапустите ИИ-приложение.

=============================================================================
ЕСЛИ НЕ РАБОТАЕТ:
=============================================================================

  1. Проверьте путь к серверу в конфиге MCP: должен указывать на server\1c-config-server.exe.
  2. Убедитесь, что в projects.json есть активный проект и в нём добавлена хотя бы одна база.
  3. Полностью перезапустите ИИ-клиент.
  4. Проверьте логи вашего ИИ-приложения.
